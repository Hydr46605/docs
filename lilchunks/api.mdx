# API Documentation

Information about future API development and current integration possibilities with LilChunks.

<Warning>
LilChunks currently does not have a public API. This page outlines planned API features and current integration methods.
</Warning>

## Current Status

LilChunks is currently focused on providing a stable, efficient chunk pregeneration experience through its command interface. **No public API is currently available** for developers to integrate with.

## Current Integration Methods

While there's no formal API, there are some ways to work with LilChunks:

### Command-Based Integration

You can integrate with LilChunks by executing commands programmatically:

```java
import org.bukkit.Bukkit;
import org.bukkit.entity.Player;

public class LilChunksIntegration {
    
    public void startPregeneration(Player player, int radius) {
        // Execute LilChunks command as player
        player.performCommand("lc start " + radius);
    }
    
    public void checkStatus(Player player) {
        // Check generation status
        player.performCommand("lc status");
    }
    
    public void stopGeneration(Player player) {
        // Stop generation
        player.performCommand("lc stop");
    }
}
```

### Console Command Integration

For server-side automation:

```java
import org.bukkit.Bukkit;

public void automatePregeneration(String worldName, int radius) {
    // Execute console commands
    Bukkit.dispatchCommand(Bukkit.getConsoleSender(), 
        "lc start " + radius + " " + worldName);
}
```

### Plugin Detection

Check if LilChunks is available:

```java
public boolean isLilChunksAvailable() {
    return Bukkit.getPluginManager().getPlugin("LilChunks") != null;
}
```

### GUI Integration

Create custom GUIs that interact with LilChunks:

```java
public class LilChunksGUI {
    
    public void openGenerationGUI(Player player) {
        // Create inventory GUI
        Inventory gui = Bukkit.createInventory(null, 27, "Chunk Generation");
        
        // Add items for different radius options
        ItemStack small = new ItemStack(Material.GRASS_BLOCK);
        ItemMeta smallMeta = small.getItemMeta();
        smallMeta.setDisplayName("Small Area (5 chunks)");
        small.setItemMeta(smallMeta);
        gui.setItem(10, small);
        
        // Handle clicks to execute commands
        player.openInventory(gui);
    }
    
    @EventHandler
    public void onInventoryClick(InventoryClickEvent event) {
        if (event.getView().getTitle().equals("Chunk Generation")) {
            Player player = (Player) event.getWhoClicked();
            
            switch (event.getSlot()) {
                case 10:
                    player.performCommand("lc start 5");
                    break;
                // Handle other options
            }
            
            event.setCancelled(true);
            player.closeInventory();
        }
    }
}
```

        // Schedule pregeneration during off-peak hours

## Workarounds and Best Practices

### Current Best Practices

<AccordionGroup>
  <Accordion title="Command-Based Integration" icon="terminal">
    **Approach**: Use command execution for integration
    
    **Benefits**:
    - Works with current version
    - Simple implementation
    - No dependency issues
    
    **Limitations**:
    - Limited control
    - No direct feedback
    - Command parsing required
  </Accordion>
  
  <Accordion title="Configuration Management" icon="gear">
    **Approach**: Modify configuration files programmatically
    
    **Benefits**:
    - Can adjust settings
    - Persistent changes
    - Server restart applies changes
    
    **Limitations**:
    - Requires restart/reload
    - File system access needed
    - No runtime validation

    - Log format dependency
